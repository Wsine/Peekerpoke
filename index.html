Workflow

  1. 切换到各自的分支

    git checkout wzy

  2. 编辑与编译

    编辑完之后，使用make编译代码，make help可以查看编译选项

  3. 提交代码

    如果使用了make编译，直接git add . 然后 git commit -m "some commit message" 即可（.gitignore 中已忽略 /build 目录的所有内容）。如果使用其他方式编译，commit 时请注意将编译生成的文件排除在外。

  4. 推送到 Github 上的分支

    git push origin wzy

  5. 最后提交 Pull Request 到 master 即可

    所有成员都有写入的权限，如果没有冲突的话也可以直接自动合并。有冲突记得解决冲突。
